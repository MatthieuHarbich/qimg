post   /images

  • Description
  • Request
  • Response

Upload an image.

Use the application/json content type to upload base64-encoded data.

Use the multipart/form-data content type to upload an image file. Send the image file as the image field.

Images cannot be larger than 2MB. (When sending base64-encoded image data, the entire JSON payload cannot be larger than 2MB.)

Quota

If you have reached the image quota (10 by default), your oldest uploaded images will be purged.

Headers

  • Authorization required (string)

    The Authorization header must contain the administrator’s bearer token. Users are not allowed access to this resource.

    Authorization: Bearer $BEARER_TOKEN

Body

Type: application/json

Example: 
{
  "data": "YXdvNHQgOHNpOGhnIHM4aHQ="
}

Type: multipart/form-data

Example: 
Content-Type: multipart/form-data; boundary=AaB03x

--AaB03x
Content-Disposition: form-data; name="image"; filename="image.png"
Content-Type: image/png

... contents of image.png ...
--AaB03x--

Body

HTTP status code 413

The uploaded image is too large.

Body

Type: text/plain

Example:

HTTP status code 415

The body of the request is in an unsupported format.

Body

Type: text/plain

Example: 
The request must have content type application/json or multipart/form-data.

HTTP status code 422

There are errors in the body of the request.

For the application/json content type, this happens if the “data” property is missing. For the multipart/form-data content type, this happens if the “image” field is not set.

Body

Type: text/plain

Example: 
The "data" property must contain the base64-encoded image data.

The "image" field is not set.

post   /tokens

  • Description
  • Request
  • Response

Create a user authentication token. This token can be used to upload and list images as a user.

Tokens are only valid for a limited time. You can specify an optional lifetime property (in seconds) which defaults to 2,592,000 (30 days). It must be at least 1 second and at most 31,536,000 seconds (365 days).

You can provide an optional name property (up to 50 characters long).

Headers

  • Authorization required (string)

    The Authorization header must contain the administrator’s bearer token. Users are not allowed access to this resource.

    Authorization: Bearer $BEARER_TOKEN

Body

Type: application/json

Example: 
{
  "lifetime": 2592000,
  "name": "foo"
}

HTTP status code 201

HTTP status code 422

There are errors in the body of the request.

This can happen if

  • the lifetime is not an integer, is less than one, or is more than 365 days;
  • the name is not a string, or longer than 50 characters.

Body

Type: text/plain

Example: 
The "lifetime" property must be an integer.

The "lifetime" property must be greater than zero.

The "lifetime" property must be less than or equal to 31,536,000 (365 days).

The "name" property must be a string.

The "name" property must not be longer than 50 characters.
